# TASK 01 — ArchAI: общая дизайн-система и технический фундамент

**Порядок:** выполнить первой.  
**Область:** общие стили и компоненты.  
**Не выполнять:** полную перестройку главной и продуктовых страниц.

## Цель

Подготовить единый визуальный и технический фундамент ArchAI, чтобы следующие страницы собирались на реальном контенте без повторного переписывания CSS и без превращения всех продуктов в один шаблон.

Перед изменениями изучить репозиторий, сохранить текущий статический стек, рабочий Cloudflare/Wrangler-конфиг и существующие URL.

## Источники истины

1. `docs/content/00_ARCHAI_MASTER.md` — глобальные правила бренда, ссылок, цен и визуала.
2. `docs/content/01_VISUAL_CORRECTIONS.txt` — последние UX-правки.
3. Существующий код — только техническая основа, не источник смыслов.

Не сокращать и не переписывать тексты страниц. Не начинать TASK 02.

## Что сохранить

Электрический синий блок первого экрана главной сохранить без ухудшения:

- цвет;
- яркость;
- насыщенность;
- движущийся контур;
- свечение;
- ощущение живого поля.

Разрешены только технические исправления, доступность и адаптивность.

## Палитра

Убрать ощущение чистого чёрного и «чёрной дыры».

Создать общие CSS-переменные для:

```css
--bg-deep;
--bg-section;
--bg-raised;
--surface-glass;
--surface-border;
--text-primary;
--text-secondary;
--text-muted;
--accent-electric;
--accent-cyan;
--accent-conifer;
--accent-violet;
--accent-sapphire;
--accent-project;
--shadow-soft;
--glow-electric;
```

Требования:

- фон — глубокий ночной синий;
- секции отличаются на 1–2 тона;
- карточки — тёмный сине-графитовый;
- основной текст — молочно-серебристый;
- вторичный — серебристо-серый;
- электрический синий только на ключевых живых элементах;
- бирюза, хвойный и мягкий фиолетовый — смысловые акценты;
- без кислотных цветов;
- без резких провалов в абсолютный чёрный.

## Типографика

Настроить единую шкалу `display`, `h1`, `h2`, `h3`, body, small, caption и navigation.

- заголовки выразительные, но не плакатные;
- не делать всё чрезмерно жирным;
- увеличить слишком мелкий текст внутренних страниц;
- ограничить длину строки;
- уменьшить гигантские пустые интервалы;
- использовать `clamp()`;
- не уменьшать мобильный текст до нечитаемого.

## Сетка

Создать общие:

- container;
- section spacing;
- compact spacing;
- двухколоночный Hero;
- асимметричную сетку;
- мобильный стек.

Не растягивать блок на пол-экрана ради двух строк.

## Header

Единый фиксированный стеклянный Header.

Логотип:

- `ArchAI`;
- `Агентство цифровых личностей`.

Меню:

- Компаньон;
- Напарник;
- Соавтор;
- Проект;
- Сайты с AI;
- Дополнительно.

Справа:

- спокойный переход к `index.html#hero-chat`;
- YouTube как неброская ссылка или тёмная кнопка;
- без ярко-красного CTA.

Мобильное меню:

- доступно клавиатурой;
- закрывается по Escape;
- без горизонтального скролла;
- фокус не теряется.

## Footer

- логотип и позиционирование;
- ссылки на продукты;
- Telegram Вика;
- YouTube Вика;
- placeholder Telegram ArchAI;
- privacy и terms.

Не показывать личный Gmail. Использовать `основательница ArchAI`, а не `основатель Светлана`.

## Кнопки

Создать варианты:

- primary;
- secondary;
- ghost;
- text-link;
- media-link;
- compact CTA.

Обязательно:

- видимый hover;
- видимый keyboard focus;
- нормальная зона нажатия;
- `noopener noreferrer` для внешних ссылок;
- не использовать кричащий красно-розовый.

## Видео

Создать переиспользуемый медиаблок:

- локальный файл;
- внешний URL/embed;
- качественная заглушка;
- фиксированное соотношение сторон;
- видео не исчезает на телефоне;
- мобильный порядок: текст → видео → действия;
- под видео компактные ссылки;
- без пустой чёрной области.

Не создавать второй большой чат вместо видео.

## Карточки продуктов

Общий визуальный компонент:

- вся карточка кликабельна;
- название важнее цены;
- 2–4 сильных результата;
- цена и сопровождение компактные;
- `Узнать больше →`;
- убрать слово `Пример`;
- доступность клавиатурой;
- hover без скачков.

Компонент общий, но содержание и композиция страниц не должны стать одинаковыми.

## Цена и сопровождение

- цена не занимает половину экрана;
- сопровождение не кислотно-зелёное;
- рядом объясняется состав и лимиты;
- сопровождение необязательно;
- внешние сервисы и превышение лимитов могут оплачиваться отдельно.

## Система живых схем

Создать лёгкий механизм без тяжёлой библиотеки:

1. центральное ядро + окружающие ноды;
2. последовательная цепочка;
3. четыре крупные ноды;
4. вертикальная мобильная версия.

Обязательно:

- видимые связи;
- световые импульсы;
- подсветка связанной группы при hover/focus;
- короткий текст внутри нод;
- подробности рядом или ниже;
- `prefers-reduced-motion`;
- хорошая мобильная производительность;
- SVG/CSS вместо тяжёлого canvas, если этого достаточно.

Не строить схемы как стопку чёрных овалов.

## Иконки

- удалить случайные эмодзи интерфейса;
- использовать единый набор тонких локальных SVG;
- не подключать тяжёлую библиотеку без необходимости;
- декоративные SVG скрыть от скринридера.

## Доступность и motion

- спокойная скорость анимаций;
- не анимировать всё одновременно;
- проверить контраст;
- focus states;
- семантические заголовки;
- `aria-label` для кнопок без текста;
- `prefers-reduced-motion`;
- не создавать фальшивый чат.

## Технические ограничения

- сохранить статическую архитектуру;
- не мигрировать на React/Vue;
- не добавлять build step без необходимости;
- не менять URL;
- не удалять Cloudflare/Wrangler-конфиг;
- не менять тексты продуктовых страниц;
- не начинать следующую задачу.

## Проверка

Проверить 1440, 1024, 768, 390 и 360 px:

- Header;
- меню;
- видео;
- карточки;
- price block;
- схемы;
- focus;
- reduced motion;
- отсутствие горизонтального скролла;
- отсутствие ошибок в консоли.

## Результат

В конце:

1. перечислить изменённые файлы;
2. описать токены и компоненты;
3. указать, что не изменялось;
4. дать результаты проверок;
5. остановиться.

**Не переходить к главной и продуктовым страницам.**
