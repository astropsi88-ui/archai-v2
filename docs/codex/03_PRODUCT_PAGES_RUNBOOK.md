# TASK 03 — ArchAI: поэтапная сборка продуктовых страниц

**Это runbook, а не одна большая задача.**

Codex выполняет **только один этап за один запуск**. Следующую страницу нельзя начинать, пока текущая не проверена и не принята.

## Порядок

1. `03A` — Компаньон.
2. `03B` — Напарник.
3. `03C` — Соавтор.
4. `03D` — Личность проекта.
5. `03E` — Сайты с AI.
6. `03F` — общий QA.

## Источники

Глобально:

- `docs/content/00_ARCHAI_MASTER.md`;
- `docs/content/01_VISUAL_CORRECTIONS.txt`;
- дизайн-система TASK 01.

Для страниц:

- `03A`: `docs/content/10_COMPANION.md`;
- `03B`: `docs/content/20_PARTNER.md`;
- `03C`: `docs/content/30_COAUTHOR.md`;
- `03D`: `docs/content/40_PROJECT_PERSONALITY.md`;
- `03E`: `docs/content/50_AI_SITES.md`.

Приоритет:

1. отдельный финальный файл страницы;
2. Master для глобальных правил, ссылок и фирменных фраз;
3. Visual Corrections;
4. существующий HTML только как техническая основа.

Если старый текст продукта в Master конфликтует с отдельным финальным файлом, использовать отдельный финальный файл.

## Общие ограничения

- менять только указанную страницу и необходимые общие компоненты;
- не переписывать другие страницы;
- не сокращать утверждённый текст до нескольких фраз;
- не заменять разные страницы одним шаблоном;
- не оставлять всё белым на чёрном;
- не делать гигантскую цену;
- не использовать кислотное сопровождение;
- убрать случайные эмодзи;
- не использовать CTA `Написать Светлане`;
- все CTA к Вику ведут на `index.html#hero-chat`;
- заказ ведёт на `https://t.me/Svetlana_itaf`;
- Telegram ArchAI не выдумывать;
- видео — реальный медиаблок или качественная заглушка;
- сохранить статическую архитектуру;
- не добавлять framework;
- не обещать безлимит;
- demo не выдавать за живой AI;
- после каждого этапа остановиться.
